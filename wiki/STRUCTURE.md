# Structure Documentation

## Repartition

- `wiki/`: documentation canonique et maintenue
- `docs/`: notes techniques complementaires + archives historiques
- `docs/archives/`: comptes rendus anciens (non normatifs)

## Fichiers cle

- Projet:
  - `README.md`
  - `CONTRIBUTING.md`
- Wiki:
  - `wiki/INDEX.md`
  - `wiki/GETTING_STARTED.md`
  - `wiki/ARCHITECTURE.md`
  - `wiki/API_REFERENCE.md`
- Guides:
  - `wiki/guides/backend/*`
  - `wiki/guides/frontend/*`
  - `wiki/guides/ops/*`

## Convention

- Toute affirmation doit correspondre au code courant
- Pas de roadmaps implicites dans les pages de reference
- Les exemples de payload doivent suivre les routes/tests actuels
